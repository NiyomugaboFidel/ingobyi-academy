# Docker deployment — Ingobyi Academy

> **Hosting on Railway?** See **[RAILWAY.md](RAILWAY.md)** instead (no Docker required).

---

The entire project runs in Docker: **PostgreSQL**, **NestJS API**, **Next.js frontend**, and **nginx** (single entry URL).

## Prerequisites

- Docker 24+
- Docker Compose v2: `sudo apt install docker-compose-plugin`
- Make (optional, for shortcuts)

Verify: `./scripts/compose.sh version`

## Quick start (in-house testing)

```bash
# 1. Create .env (auto-generates secrets)
make docker-init

# 2. Start everything
make docker-up

# 3. Load demo users (first time only)
make docker-seed
```

Open **http://localhost** (or your `PUBLIC_URL`).

**Demo login:** `super@ingobyi.com` / `password123`

## Architecture

```
                    ┌─────────────┐
   Browser ────────►│ nginx :80   │
                    └──────┬──────┘
              ┌────────────┼────────────┐
              ▼            ▼            │
         web :3000    api :3001        │
         (Next.js)    (NestJS)         │
              │            │            │
              └────────────┼────────────┘
                           ▼
                    postgres :5432
                    (internal only)
```

| Path | Service |
|------|---------|
| `/` | Next.js frontend |
| `/api/*` | NestJS REST API |
| `/socket.io/*` | WebSocket (chat, presence) |

## Configuration (`.env`)

Copy template: `cp .env.staging.example .env` or run `make docker-init`.

| Variable | Required | Notes |
|----------|----------|-------|
| `PUBLIC_URL` | Yes | URL users open, e.g. `http://192.168.1.50` |
| `FRONTEND_URL` | Yes | Same as `PUBLIC_URL` |
| `POSTGRES_PASSWORD` | Yes | Auto-generated by `docker-init` |
| `JWT_SECRET` | Yes | Auto-generated (min 16 chars) |
| `REFRESH_SECRET` | Yes | Auto-generated |
| `HTTP_PORT` | No | Default `80` |
| `SHOW_DEMO_CREDENTIALS` | No | `true` for testers, `false` for production |
| `ENABLE_SWAGGER` | No | API docs at `/api/docs` |
| `CLOUDINARY_*` | No | Image uploads |
| `SMTP_*` | No | Email OTP |

**LAN deployment:** set `PUBLIC_URL=http://YOUR_SERVER_IP` and leave `COOKIE_DOMAIN` empty.

## Moving to public production

1. `NODE_ENV=production`
2. `SHOW_DEMO_CREDENTIALS=false`
3. `ENABLE_SWAGGER=false`
4. HTTPS + `COOKIE_SECURE=true`
5. Strong secrets (never use seed passwords)
6. Do **not** run `docker-seed` in production
7. Configure SMTP + Cloudinary
8. Back up `postgres_data` volume regularly

## Troubleshooting

```bash
docker compose ps
docker compose logs api
docker compose logs web
curl http://localhost/api/health
```

| Problem | Solution |
|---------|----------|
| Port 80 busy | Set `HTTP_PORT=8080`, `PUBLIC_URL=http://localhost:8080` |
| 502 from nginx | Wait ~2 min for first build; check `docker compose logs api web` |
| Auth cookies fail | `PUBLIC_URL` must match browser URL; empty `COOKIE_DOMAIN` for IP |
| No users | `make docker-seed` |
